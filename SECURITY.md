# Security Policy

## Supported versions

This project is provided as-is. Security fixes are applied on a best-effort basis to the latest `main`.

## Reporting a vulnerability

Please **do not** open a public issue for secrets or exploitable bugs.

- Prefer a private GitHub Security Advisory on the repository, or
- Contact the maintainer via GitHub profile contact options.

Include:

- Description and impact
- Steps to reproduce
- Affected commit / release if known

Do not include live API keys in reports.
